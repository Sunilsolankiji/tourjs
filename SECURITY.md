# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in tourjs, please report it by emailing security@example.com instead of using the issue tracker.

Please include the following information:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

We will acknowledge your report within 48 hours and provide a detailed response within 5 days.

## Security Updates

We take security seriously and will release patches for security vulnerabilities as soon as possible.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Best Practices

When using tourjs in your application:

1. Keep the library updated to the latest version
2. Review the changelog for security updates
3. Test updates in a staging environment first
4. Report any suspicious behavior or vulnerabilities

## Third-party Dependencies

tourjs is a zero-dependency library, which means it doesn't rely on any external packages. This reduces potential security risks from dependency vulnerabilities.

Thank you for helping keep tourjs secure!
